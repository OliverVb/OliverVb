## 🛡️ Segurança & Boas Práticas

Princípios que aplico em todos os projetos:

- 🔑 **Secrets nunca em código** — gerenciamento com HashiCorp Vault e AWS Secrets Manager
- 🔍 **Scanning contínuo de dependências** — Dependabot, Trivy e Snyk integrados aos pipelines
- 📄 **`SECURITY.md`** presente em repositórios sensíveis, com política de divulgação responsável
- ✍️ **Commits assinados com GPG** para garantir autenticidade e integridade
- 🚫 **`.gitignore` rigoroso** — nenhum `.env`, chave privada ou certificado versionado
- 📦 **Imagens Docker minimalistas** — base `distroless` ou `alpine`, com superfície de ataque reduzida

> ⚠️ **Encontrou uma vulnerabilidade em algum dos meus projetos?**
> Reporte de forma privada via **Security Advisory** do repositório afetado (aba *Security* → *Report a vulnerability*), e **não** em issues públicas.
